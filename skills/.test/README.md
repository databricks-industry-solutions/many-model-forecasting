# skill-test

Testing framework for evaluating Databricks skills, with a focus on the
Many-Model Forecasting (MMF) skill.

## Quick start

```bash
cd .test
uv sync --extra dev          # unit tests only
uv sync --extra tier1        # + tier 1 mock-infrastructure tests
uv sync --extra all          # everything
```

## Test tiers

### Unit tests (`tests/test_scorers.py`) — 17 tests

Fast, zero-infrastructure tests that validate the framework's own logic.

| Area | What it covers |
|------|----------------|
| `TestDetectSkillsFromPrompt` | Skill detection from user prompts (case-insensitive keyword matching) |
| `TestSkillTriggers` | SKILL_TRIGGERS configuration completeness and format |
| `TestExtractCodeBlocks` | Python/SQL code block extraction from LLM responses |
| `TestVerifyPythonSyntax` | Python syntax validation of extracted code |
| `TestVerifySqlStructure` | SQL structure checks (balanced parens, statement detection) |

```bash
uv run pytest tests/test_scorers.py -v
```

### Tier 1 tests (`tests/tier1/`) — 14 tests

Skill-logic tests that run the MMF agent against a **DuckDB mock backend**.
SQL generated by the agent is transpiled from Databricks dialect to DuckDB
via SQLGlot, so no live Databricks workspace is needed. However, the agent
loop itself calls **Claude Sonnet on Databricks FMAPI**, so a
`DATABRICKS_TOKEN` is required.

| File | Tests | What it covers |
|------|-------|----------------|
| `test_explore_data.py` | 9 | Table discovery (SHOW TABLES, DESCRIBE), column mapping, profiling queries (COUNT, MIN/MAX, DISTINCT), data quality checks (NULLs, negatives), weekly/monthly date alignment in CREATE TABLE |
| `test_setup_cluster.py` | 2 | CPU cluster config (i3.xlarge, CPU runtime) and GPU cluster config (g5.12xlarge, GPU runtime, single-node) |
| `test_run_mmf.py` | 3 | Notebook parameter generation for daily, weekly, and monthly frequencies |

**Infrastructure:** `DATABRICKS_TOKEN` + `DATABRICKS_HOST` (defaults to
`fevm-retail-webapp-commerce.cloud.databricks.com`).

```bash
export DATABRICKS_HOST="https://..."
export DATABRICKS_TOKEN="dapi..."
uv run pytest tests/tier1/ -v -m tier1
```

**How it works:**

1. The `conftest.py` creates an in-memory DuckDB database seeded with test
   data (50 daily series, 10 weekly, 10 monthly, plus metadata tables).
2. Skill markdown files from `databricks-skills/many-model-forecasting/` are
   loaded as system prompts.
3. The agent runner sends prompts to Claude Sonnet, intercepts tool calls,
   and executes them against DuckDB mock handlers.
4. Tests assert on the SQL statements the agent generated and the final
   response content.

### Evaluation framework (`scripts/run_eval.py`)

LLM-judge-based evaluation that scores agent responses against ground truth
test cases defined in YAML.

| Component | Location | Purpose |
|-----------|----------|---------|
| Ground truth | `skills/many-model-forecasting/ground_truth.yaml` | 5 test cases (explore-data, setup-cluster CPU/GPU, run-mmf local/foundation) with expected facts, patterns, and guidelines |
| Manifest | `skills/many-model-forecasting/manifest.yaml` | Scorer configuration, quality gates, trace expectations |
| Candidates | `skills/many-model-forecasting/candidates.yaml` | Recorded agent responses to evaluate (currently empty) |

**Scorers** (defined in `src/skill_test/scorers/`):

- `python_syntax` — validates Python code blocks parse correctly
- `sql_syntax` — validates SQL structure (balanced parens, keywords)
- `pattern_adherence` — checks that required patterns appear in the response
- `no_hallucinated_apis` — detects made-up API calls
- `expected_facts_present` — verifies factual claims against ground truth
- `guidelines_from_expectations` — LLM-judge scoring against custom guidelines
- `skill_routing_accuracy` / `routing_precision` / `routing_recall` — measures
  whether the correct skill was detected from the user prompt

```bash
python scripts/run_eval.py many-model-forecasting
python scripts/run_eval.py many-model-forecasting --test-ids mmf_20260305_explore_data
```

## Source layout

```
.test/
├── pyproject.toml
├── scripts/                        # CLI entry points
│   ├── _common.py
│   └── run_eval.py
├── skills/                         # Per-skill test data
│   └── many-model-forecasting/
│       ├── manifest.yaml           # Scorer config & quality gates
│       ├── ground_truth.yaml       # Expected responses
│       └── candidates.yaml         # Recorded responses to evaluate
├── src/skill_test/                 # Framework library
│   ├── cli/                        # CLI commands
│   ├── config.py                   # SkillTestConfig, QualityGates
│   ├── dataset.py                  # EvalRecord, YAML/UC dataset sources
│   ├── fixtures/                   # Test fixture setup
│   ├── grp/                        # Generate-Review-Perfect pipeline
│   │   ├── executor.py             # Code block extraction & validation
│   │   ├── diagnosis.py            # Failure diagnosis
│   │   ├── reviewer.py             # Response review
│   │   └── pipeline.py             # GRP orchestration
│   ├── mock/                       # DuckDB mock backend (tier 1)
│   │   ├── duckdb_backend.py       # SQL transpilation & execution
│   │   ├── mock_tools.py           # OpenAI-format tool definitions
│   │   ├── agent_runner.py         # Multi-turn agent loop
│   │   └── fixtures.py             # Pre-canned non-SQL responses
│   ├── optimize/                   # Prompt optimization (GEPA)
│   ├── runners/                    # Evaluation & comparison runners
│   ├── scenarios/                  # Synthetic data generators (11 scenarios)
│   │   ├── generators.py           # Time-series data generators
│   │   └── scenarios.py            # Named scenario configurations
│   ├── scorers/                    # Evaluation scorers
│   │   ├── universal.py            # Deterministic scorers
│   │   ├── routing.py              # Skill routing scorers
│   │   └── dynamic.py              # LLM-judge scorers
│   └── trace/                      # MLflow trace parsing & integration
└── tests/                          # Pytest test suites
    ├── test_scorers.py             # Unit tests (17 tests)
    └── tier1/                      # Tier 1 mock tests (14 tests)
        ├── conftest.py             # Fixtures (DuckDB, OpenAI client, skill prompts)
        ├── test_explore_data.py    # explore-data skill tests
        ├── test_setup_cluster.py   # setup-cluster skill tests
        └── test_run_mmf.py         # run-mmf skill tests
```

| `databricks` | databricks-sdk | Real workspace integration |
| `optimize` | gepa, tiktoken | Prompt optimization |
| `all` | Everything above | Full development setup |
